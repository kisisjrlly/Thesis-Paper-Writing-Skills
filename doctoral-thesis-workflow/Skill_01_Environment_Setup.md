# Skill 01: Environment Setup

## 目标

自动完成博士论文流水线的基础环境准备，确保 Agent 后续可稳定执行检索、解析与编译任务。

## 输入

- 操作系统信息（macOS / Linux / Windows）
- 当前工作区路径
- 是否已有文献管理工具（Zotero）
- 是否允许安装开发工具（如 `uv`）

## 输出

- 环境检查报告（通过 / 缺失项 / 修复动作）
- MCP 配置文件（如 `.mcp.json`）
- 工具链状态结论（可进入 Skill 02）

## 执行步骤（Agent SOP）

1. **前置扫描**
   - 检查系统是否可用 `git`、`python`、`xelatex`、`node`（若 Zotero MCP 依赖 Node）。
   - 检查是否已安装 `uv`。
2. **安装或修复 `uv`**
   - 若不存在，按平台静默安装并二次验证版本。
3. **配置文献检索链（Zotero MCP）**
   - 检查本机是否可访问 Zotero 数据源。
   - 初始化 `zotero-mcp-server` 配置。
4. **写入工作区 MCP 配置**
   - 在仓库根目录生成或更新 `.mcp.json`。
   - 写入最小可用 server 列表与启动参数。
5. **连通性验证**
   - 执行一次最小检索请求（例如按关键字查 1 条文献）。
   - 若失败，输出定位信息（认证、端口、路径、权限）。

## 失败处理

- `uv` 安装失败：切换镜像源 / 管理员权限提示 / 降级到系统 Python 环境。
- Zotero MCP 不可达：检查本地客户端状态、插件配置、端口冲突。
- `.mcp.json` 写入失败：检查路径权限，自动回退到用户目录配置。

## 完成判据（Definition of Done）

- `uv` 可调用且版本可见。
- MCP 配置文件存在且格式合法。
- 至少一次检索调用成功。

## 交付格式（建议）

- 环境状态表
- 自动修复日志
- 下一步建议（进入 `Skill_02_Asset_Extraction.md`）
