# patterns-vocabulary: 词汇黑名单与术语恰当性

所有示例均为通用造句，不针对任何具体学科。实际应用时结合语境灵活判断。

---

## A. 词汇黑名单

### A1. 套话过渡词（硬命中，必改）

**黑名单**（含所有变体）：
- 值得注意的是 / 需要指出的是 / 不难看出 / 可以看出 / 可以发现 / 显而易见
- 特别地 / 进一步地 / 进一步而言 / 此外 / 与此同时
- 综上所述 / 总而言之 / 通过上述分析可知 / 由此可见
- 本文旨在 / 本节将 / 本章将 / 本节首先 / 接下来
- 首先……其次……最后…… / 一方面……另一方面……
- 不仅……而且…… / 不仅如此

**改写方式**：直接删除，或让下一句自然承接上一句。连接词绝大多数情况下不需要显性写出，语序本身就能表达逻辑。

**错例**：
> 综上所述，本方法有效提升了系统性能。此外，在复杂工况下也表现出鲁棒性。

**改**：
> 本方法提升了系统性能，在复杂工况下也保持鲁棒。

### A2. 口语化/对话化表达

**黑名单**：
- 我们来看看 / 接下来聊聊 / 下面说说
- 我们觉得 / 我们发现 / 我们认为（当没有做对比实验支撑时）
- 能够 / 可以（主语能力已经明确时）
- 值得一提的是

**改写**：
- "我们发现 X" → "实验结果表明 X" / "X" 本身直接陈述
- "该算法能够处理 XX" → "该算法处理 XX"
- "我们觉得该方法更合适" → 若有数据：给出数据并下结论；若无，删掉整句

### A3. 空洞修饰与升华词

**黑名单**（命中即必改）：
- 根源 / 基石 / 奠定……基础 / 至关重要 / 具有重要意义
- 开辟道路 / 开创性 / 突破性 / 前所未有
- 深层成因 / 精准识别 / 根本性
- 显著（未接具体数字时）/ 高效 / 有效（作形容词悬空时）
- 某某的能力 / 某某的品质 / 某某的闭环 / 某某的逻辑（空洞用法）
- 有效平衡 / 有效应对 / 有效提升

**改写方式**：用数字或具体机理替代形容词。"显著提升"要么改成"提升 12%"，要么删。"有效平衡"几乎可以直接删。

**错例**：
> 该网络具有深层的特征提取能力，显著提升了识别精度，为后续任务奠定了坚实基础。

**改**：
> 该网络的识别精度在测试集上达到 92.3%，较基线方法高 3.1 个百分点。

### A4. 公文腔

**黑名单**：为了显"学术"把"是"改"系"、把"旨在"改"拟"、把"以"改"拟以"。

**改写方式**：保持当代学术书面语就好。"旨在"本身可以用，但别改成"拟"；"是"就写"是"。

### A5. 名词化动词堆砌（"进行+名词"）

**黑名单**：
- 进行分析 / 进行优化 / 进行建模 / 进行更新 / 进行验证 / 进行对比
- 进行研究 / 进行考虑 / 进行比较

**改写**：还原动词本身。

| 错 | 对 |
|---|---|
| 对数据进行分析 | 分析数据 |
| 对模型进行优化 | 优化模型 |
| 对误差进行补偿 | 补偿误差 |

**例外**：少数固定搭配（如"进行实验"）可保留，但如果一段里"进行 X"出现 3 次以上，必有冗余。

### A6. 归因套话

**黑名单**：原因在于…… / 其原因在于…… / 之所以……是因为……（当解释简单时）

**改写**：
- "原因在于 X" → "这是因为 X"
- 解释一句话能说清时直接组句，不用归因结构

### A7. 其他常见冗余副词

**慎用**（多数可删）：非常、十分、极大地、高度地、较好地、很好地、比较

**原则**：形容词前的副词通常没加任何信息。"较好地完成了任务"等于"完成了任务"。"比较准确"不如"准确率 87%"。

## E. 术语恰当性

### E1. 两条核心测试（最重要）

判定一段话是否适合毕业论文，用这两个测试：

1. **删除测试**：这句话删掉或换成更短表达，意思有损失吗？没损失就是 AI 填充，删。
2. **范例测试**：这句话有没有可能出现在用户指定的范例毕业论文里？不可能就换。

这两条比下面任何具体黑名单都更本质。先过这两关。

### E2. 学术会议/论文口吻的黑名单

这些表述在会议论文和博客里很常见，但放进毕业论文就不合适：

- 留下三处明显空白 / 能力缺口 / 长期被低估的环节
- 贡献仍为零（或类似极端表达）
- 机器精度量级 / 子空间正交性在数据上得到对应（这种"拗口的精确"）
- 固有的 XX 下降率 / 固有的 XX 性质（把普通性质说得像定理）
- 弥合了……之间的能力缺口 / 弥合……差距
- 这一"整合"本身不是贡献 / 这不仅仅是 A，更是 B
- 一方面……另一方面……（用在简单对比上时）
- 在 X 领域和 Y 领域都有明显的问题（泛化的空话）

**判断原则**：如果一句话删掉之后，技术内容一点都不少，那它就是空话。

### E3. 自造术语作为小节标题或关键概念

研究者思维倾向"越精准越好"，喜欢造"基于 X 的 Y 跨层 Z"这样的复合术语。但毕业论文更看重使用**学界公认的标准术语**。

**原则**：
- 同一概念有公认术语时，优先用公认术语
- 确需造新术语时，第一次出现要给出明确定义，并解释为什么现有术语不够
- 不要在小节标题里放自造术语（见 patterns-structure.md D2）

### E4. 刻意拔高系统难度

写引言/场景描述时，不要为了显得研究有必要而过度描述系统难度，比如"对精度要求极其苛刻"、"涉及多级精密传动"这类。答辩老师会追问具体数据，顶不住就是减分项。

**原则**：给具体指标或直接不说。"要求跟踪误差小于 0.1°" > "对精度要求极其苛刻"。

### E5. 缩写的引入

- 缩写第一次出现时必须给出中文全称和英文原名：如"长短期记忆网络（Long Short-Term Memory, LSTM）"
- 摘要里避免堆缩写
- 自定义缩写（非公认）尤其要慎重引入
- 全文首次出现后才使用缩写，不要在标题或摘要首句里直接用未解释的缩写

### E6. 当前学术界公认 vs 半口语术语

写作或审校时遇到可疑术语，查一下是不是学界公认译名：
- 翻译类专业术语以本领域权威期刊的通行译名为准，不要自创译名
- 遇到看似多种译法的词，查一下该领域高引中文论文的用法，选其中主流一种
- 避免把英文短语直译拼成看起来像术语的组合（这类词在不同学科含义差异大，容易引起误读）

## 使用建议

写作模式下：每写完一段，先按这个文件的 A1-A7 过一遍词汇，再用 E1 的两个测试整体过。

审校模式下：把输入按 A1-A7 扫硬命中，E2-E6 做语境判断。命中 A 类的标"必改"，命中 E 类的根据程度标"必改/建议"。
