# Skill 01: Environment Setup

## 目标

检查博士论文流水线的基础环境，给出最小修复方案，确保后续可稳定执行检索、解析、写作与编译任务。

## 输入

- 操作系统信息（macOS / Linux / Windows）
- 当前工作区路径
- 是否已有文献管理工具（Zotero）
- 是否允许安装开发工具（如 `uv`）

## 输出

- 环境检查报告（通过 / 缺失项 / 修复动作）
- 可选 MCP 配置建议（如 `.mcp.json`）
- 工具链状态结论（可进入 Skill 02）

## 执行步骤（Agent SOP）

1. **前置扫描**
   - 检查当前工作区、Git 状态、Python、XeLaTeX、BibTeX/Biber、Node（如需 MCP）、`uv`。
   - 检查论文模板入口文件（例如 `main.tex`）和参考文献文件是否存在。
2. **能力分级**
   - `ready`：可直接抽取资产并编译模板。
   - `partial`：可写作或抽取，但缺编译或文献链路。
   - `blocked`：缺少源文件、模板或 Python 等关键条件。
3. **修复建议**
   - 只输出明确命令和影响范围。
   - 用户明确要求后再安装工具、修改 MCP 配置或写入全局配置。
4. **文献检索链（可选）**
   - 若用户需要 Zotero/MCP，检查本机可访问性并给出最小配置。
   - 验证失败时输出认证、端口、路径、权限等定位信息。
5. **编译烟测**
   - 若已有 `main.tex`，优先运行一次最小 XeLaTeX 编译或 dry-run 风格检查。
   - 保存编译日志位置，避免覆盖用户已有输出。

## 失败处理

- 缺 `uv`：若本流程只需标准库脚本，可降级到系统 Python 环境。
- Zotero MCP 不可达：检查本地客户端状态、插件配置、端口冲突。
- `.mcp.json` 写入失败：检查路径权限；不要自动回退到用户目录配置，先征求用户确认。

## 完成判据（Definition of Done）

- 必需工具状态清楚，缺失项有可执行修复命令。
- 若已提供模板，入口文件和编译方式明确。
- 若启用文献链路，至少一次检索调用成功。

## 交付格式（建议）

- 环境状态表
- 自动修复日志
- 下一步建议（进入 `Skill_02_Asset_Extraction.md`）
