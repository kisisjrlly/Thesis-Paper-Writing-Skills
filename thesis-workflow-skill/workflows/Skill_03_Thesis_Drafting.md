# Skill 03: Thesis Drafting (Core)

## 目标

将小论文资产映射到博士论文模板，按章节渐进式扩写并保持可编译。

## 输入

- 博士论文模板（`.cls`、主 `.tex`、章节骨架）
- Skill 02 输出的资产 JSON
- 术语约束与写作风格约束

## 强制术语锁定（Term Lock）

- 全局统一使用：`drone`
- 现实动机统一表述：`real-world condition constraints`
- 从控制变量过渡到理论解释时，优先强调：`physics-informed priors`

## 输出

- 按章节增量生成的论文 `.tex`
- 每章 claim-evidence 对齐表
- 编译日志与修复记录

## 执行步骤（Agent SOP）

1. **大纲映射**
   - 将资产映射到“问题定义—方法—实验—讨论”章节结构。
2. **章节级循环生成（禁止一口气全量）**
   - 每轮仅处理一个章节。
   - 先生成段落骨架，再填充公式、图表与引用。
3. **叙事扩写策略**
   - 将小论文精简表达扩写为博士论文背景、动机与方法细节。
   - 保留数学严谨性，不随意改写符号系统。
4. **自动编译闭环**
   - 每章完成后触发 `xelatex` 编译。
   - 若失败，解析 `.log` 定位错误并自动修复。
   - 直到“本章增量可编译”再进入下一章。
5. **一致性检查**
   - 术语一致性（drone / constraints / priors）。
   - 图表、公式、引用跨章编号一致。

## 失败处理

- 编译报错（缺包、引用、编码）：优先最小修复，不大改结构。
- 章节过长导致上下文退化：分节切片写作，并回填摘要段。
- 证据不足：自动下调断言强度，标记“待补实验”。

## 完成判据（Definition of Done）

- 全文章节可连续编译。
- 关键结论均有实验或理论证据支撑。
- 术语锁定策略在全稿满足一致性。
